MBPP files for Code-GRPO

Files generated by `scripts/prepare_mbpp_codegrpo.py`:

- `mbpp_sanitized_codegrpo_train.jsonl`: MBPP sanitized train split converted to Code-GRPO format.
- `mbpp_sanitized_codegrpo_validation.jsonl`: MBPP sanitized validation split converted to Code-GRPO format.
- `mbpp_sanitized_codegrpo_test.jsonl`: MBPP sanitized test split converted to Code-GRPO format.
- `mbpp_sanitized_codegrpo_trainval.jsonl`: train + validation combined.
- `mbpp_sanitized_codegrpo_small64.jsonl`: first 64 converted training examples for small-sample training.
- `mbpp_sanitized_codegrpo_summary.json`: conversion summary, including skipped examples and reasons.

Current runtime contract:

- Every task asks the model to implement `solve(x)`.
- If the original MBPP function had one positional argument, `solve(x)` receives that argument directly.
- If it had multiple positional arguments, `solve(x)` receives a Python list of positional arguments in order.
- If kwargs are present, `solve(x)` receives `{"args": [...], "kwargs": {...}}`.

Recommended first training config on server:

- `codegrpo_train_qwen7b_vllm_mbpp_small.yaml`

Typical server command:

```bash
cd ~/autodl-tmp/CODE-GRPO/trl
python scripts/prepare_mbpp_codegrpo.py --output-dir data --small-train-size 64
python -m trl.cli.main code_grpo --config codegrpo_train_qwen7b_vllm_mbpp_small.yaml
```
