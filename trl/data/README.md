MBPP files for Code-GRPO

Files generated by `scripts/prepare_mbpp_codegrpo.py`:

- `mbpp_sanitized_codegrpo_train.jsonl`: 70% train split after converting all supported MBPP sanitized examples.
- `mbpp_sanitized_codegrpo_validation.jsonl`: 15% validation split.
- `mbpp_sanitized_codegrpo_test.jsonl`: 15% test split.
- `mbpp_sanitized_codegrpo_trainval.jsonl`: train + validation combined (85%) for cases where a larger training pool is needed.
- `mbpp_sanitized_codegrpo_small64.jsonl`: first 64 converted training examples for small-sample training.
- `mbpp_sanitized_codegrpo_summary.json`: conversion summary, including skipped examples, final split counts, and split seed.

Current runtime contract:

- Every task asks the model to implement `solve(x)`.
- If the original MBPP function had one positional argument, `solve(x)` receives that argument directly.
- If it had multiple positional arguments, `solve(x)` receives a Python list of positional arguments in order.
- If kwargs are present, `solve(x)` receives `{"args": [...], "kwargs": {...}}`.

Recommended first training config on server:

- `configs/train/codegrpo_train_qwen7b_vllm_mbpp_small.yaml`

Typical server command:

```bash
cd ~/autodl-tmp/CODE-GRPO/trl
python scripts/prepare_mbpp_codegrpo.py --output-dir data --small-train-size 64 --train-ratio 0.70 --validation-ratio 0.15 --test-ratio 0.15 --split-seed 42
python -m trl.cli.main code_grpo --config configs/train/codegrpo_train_qwen7b_vllm_mbpp_small.yaml
```
